How to Write Language Agnostic Code

Introduction

This guide is intended to teach developers how to write code that is easily portable among a subset of modern, widely-used programming languages. This is a vital skill to have in the early stages of working on a project, as writing language-agnostic code reduces technical debt. The developer is then granted much more freedom and versatility in what language they continue their development in, as it does not take as much work to port generic code as it does idiomatic code.
This guide starts with the basics of several languages in wide use (namely, C++, Java, JavaScript, Python, and Ruby, which I have enough experience in to discuss), and begins by outlining the features that make them Turing complete. The guide will then expand on the features that they share, and their more advanced features, which may differ greatly.

Turing Completeness

For a programming language to be Turing complete, it must allow for conditional branching and the use of arbritrary amounts of memory. Almost all languages allow for conditional branching in at least the form of if/else statements, which are implemented in a fairly standard way.
C++, Java and JavaSript all have identical syntax for this: 

if (condition1) {
    executeSomeCode();
}
else if (condition2) {
    executeOtherCode();
}
else {
    executeThis();
}

This code will work identically in all three languages, as they are all directly based off of the syntax of C. Python and Ruby, which have different keywords and different syntax for blocks, are written slightly differently:
Ruby: 

if condition1 
    executeSomeCode();
elsif condition2 
    executeOtherCode();
else 
    executeThis();
end

Python:

if condition1:
    executeSomeCode();
elif condition2: 
    executeOtherCode();
else:
    executeThis();

It can therefore be seen that if/else statements are inherently language agnostic, both due to their ubiquity and similar syntax in common programming languages. The use of arbritrary memory, however, is less than guaranteed, as no computer can possibly offer a program infinite memory use. As this requirement cannot be reasonably applied to any language, and is in all cases ignored when evaluating a language for Turing completeness, we will instead replace it with 'offers manipulation of memory'. What other features are shared widely enough to be language agnostic?

comments, variables, atomic types (basics)

comments are all the same, assigning variables is mostly the same apart from the increment operator which is omitted from Ruby and Python (but trivially switched from one to the other via regex). keywords with assigning are different- ruby doesn't do consts (just warnings), C++ and Java have STATIC typing, leading to -> The latter languages are dynamically typed, and allow things like assigning a different type to a variable, which the others don't- avoid this! not language agnostic, leads to errors.
